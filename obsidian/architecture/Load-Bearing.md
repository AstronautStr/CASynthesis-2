# Load-Bearing решения (не менять молча)

Четыре решения, которые нельзя молча реверсировать — только явно с обоснованием.

---

## 1. Перцептивные признаки, не эмбеддинг
Маппинг строится на **интерпретируемых перцептивных признаках** формы (или на автоматических преобразованиях, дающих осмысленный физический смысл — как Laplace). Чёрный ящик/латентные оси — только вторичный слой уникальности.

*Почему:* чтобы «перцептивно корректно» смэппить признак в параметр синта, нужно знать, что признак означает. Latent axes риск активного мисматча.

## 2. Единица сонификации = перцептивный объект
Каждый связный объект поля → один голос/партиал. Поле = аудиальная сцена (Bregman ASA / Gestalt). → [[Sonification-Unit]]

## 3. Синтезатор, не секвенсор
Ось «размер/высота» объекта = гармонический номер одной карьерной ноты (или привязка мод к f0). Не отдельные ноты.

*Расширено 2026-06-15:* «гармоники одной ноты» → «произвольный спектр, привязанный по высоте к f0». Фрейминг синтезатора сохранён. → [[Frequency-Framing]]

## 4. Непрерывный постоянный звук
Фазо-когерентные партиалы, глайд амплитуд. Никаких per-generation пульсов. Кросс-фейд наборов мод — амплитудный (сохраняет непрерывность). → [[Bip-Topology-Change]]
