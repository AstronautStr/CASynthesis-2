## 2026-06-15 — Discover-first: форма объекта → весь вектор амплитуд гармоник
**Decision:** Заменить `size → одна гармоника k` на маппинг `форма объекта (бинарная
сетка) → вектор амплитуд гармоник 1..K`, где вектор получается **автоматическим**
преобразованием формы (первый кандидат — магнитудный 2D-FFT). Аддитивный
фазо-когерентный движок (§F) не меняется — меняется только источник амплитуд.
Эксперимент ведём на фиксированном окне **8×8** и зоопарке мелких осцилляторов
(блинкер, маяк, жаба, часы); цель — _открыть_ (discover) маппинг, дающий различимые
и метрически согласованные тембры, а не _сконструировать_ (craft) его вручную.

**Rationale:** Текущее `size → k` (а) систематически сажает звук в верх спектра
(статистика Жизни: объекты мелкие) → писк при умирании поля, (б) внутри голоса
оставляет чистый синус → «аддитивный набор синусов», звук одинаковый. Ручной
по-фичевый маппинг рискует долгим блужданием без гарантии уникальности.
Автоматическое преобразование даёт уникальность + сохранение метрики «по
определению» (как эмбеддинг), но без обучения и без чёрного ящика. Магнитудный
2D-FFT вдобавок **translation-invariant** (`|FFT(сдвиг)| = |FFT|`) → выполняет
вводную «позиция участка на поле не влияет на тембр» буквально и бесплатно.

**Отношение к load-bearing №1 (§C):** это НЕ отмена перцептивного соответствия.
§C сам называет автоматический/эмбеддинг-слой «вторичным слоем уникальности,
отложенным»; мы осознанно выводим его на первый план как исследовательский этап.
Перцептивное соответствие сохраняется как цель и вернётся слоем интерпретации/
чистки поверх найденного ландшафта (discover → потом, при нужде, craft).

**Implications for implementation:**
- Маппинг должен быть **сменным модулем** (плагин): форма → вектор амплитуд. Движок
  один, алгоритмов маппинга будем перебирать несколько (см. брейнсторм ниже / в логе).
- Открытый под-выбор внутри 2D-FFT: **линеаризация 2D-спектра в 1D-гармоники**
  (радиально по модулю частоты / зигзаг как в JPEG / др.) — задаёт характер тембра.
- Окно 8×8, объект центрируется/обрезается в окно (для FFT точное место не важно).
- Критерий успеха: 4 осциллятора различимы на слух И перцептивно близкие формы
  звучат ближе (блинкер↔жаба ближе, чем блинкер↔маяк).
- Архитектурный вопрос (режим приложения vs отдельный стенд) вынесен в
  `questions.md` для Разработчика.

**Уточнение (2026-06-15, по ходу сессии):** не слушаем по одному — реализуем
несколько маппингов сразу и устраиваем **batch-прослушивание** матрицы
`осциллятор × маппинг`. Решение о тембре принимается на слух, не по теории.

**Очередь маппингов (по возрастанию сложности реализации — ориентир, Разработчик
оценит точно; все выдают вектор амплитуд гармоник в тот же движок):**
1. **2D-FFT магнитуда** — `np.fft.fft2`+`abs`, тривиально. Translation-invariant.
   Ухом: «спектр узора».
2. **Уолш–Адамар** — `scipy.linalg.hadamard(64)` @ flatten, тривиально. Ухом:
   жёсткий «цифровой» родственник FFT.
3. **Случайная проекция** — фикс. `np.random.randn(K,64)`, тривиально. Контроль/
   baseline: насколько вообще трудно получить «разное».
4. **Спектр Лапласиана (граф соседства клеток)** — `scipy.sparse.csgraph.laplacian`
   + `eigvalsh`, легко. Инвариант к позиции/повороту/отражению. Ухом: резонанс
   формы, «как если по ней стукнуть» — физически осмысленный, выше шанс «красиво».
5. **Гранулометрия** — цикл `ndimage.binary_opening`, легко-средне. Ухом: мелкие
   детали формы = ярче.
6. **Zernike-моменты** — полиномы руками или `mahotas`, средне. Инвариант к повороту.
7. **Банк Габора** — банк ядер + свёртка, средне.
8. **PCA по зоопарку форм** — код лёгкий, но нужен набор форм + ручная привязка осей
   к звуку → отдельная морока, делать последней.

Дёшево реализуемы и стоит включить в первый прогон: пп. 1–5. Перцептивный приоритет
на слух: 2D-FFT и спектр Лапласиана (п. 4).
