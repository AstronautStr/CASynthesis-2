## Implemented

- **Живые крутилки тембра Laplace (2026-06-16)** — три экранных слайдера (мышь) в
  тулбаре `gol_life_synth_laplacian.py`, меняются на лету без рестарта (реализация
  реквеста questions.md «Живые крутилки», решение decisions.md 2026-06-16):
  - **spread** (0.00–1.00, дефолт 0): выбор мод от «n низших подряд» до «децимация по
    всему спектру λ вверх». Новая чистая функция `casynth_core._select_modes`; параметры
    `map_laplacian(patch, f0, n, spread=0.0, alpha=1.0)` — обратно-совместимы (стенд зовёт
    без них → бит-в-бит как было). Низшая взятая мода ВСЕГДА = f0 (привязка к несущей,
    load-bearing) при любом spread.
  - **alpha** (0.00–2.00, дефолт 1): спад `1/i^α`. α=1 = старое `1/i`; α→0 плоско/ярко;
    α>1 темнее. Частоты не трогает.
  - **release** (20–4000 мс, дефолт 50): длина хвоста моды развязана от атаки кросс-фейда.
    `SlotPool.update(voices, amp_cur, release_chunks)` берёт live `release_chunks`; длина
    фиксируется per-slot (`_release_len`) на старте release → поворот крутилки в полёте не
    рескейлит идущий хвост. Атака осталась 1 чанк. Мёртвые `FADE_MS`/`FADE_CHUNKS` удалены,
    заменены `RELEASE_MS_*`.
  - Тесты: `tests/test_casynth_core.py` +4 (backward-compat дефолтов, n=truncation-only,
    spread держит f0 и достаёт верх, alpha-яркость) — 14/14 PASS. Headless-smoke прототипа
    и стенда чисты; функциональный проб движка с новыми крутилками — звук считается.
  - **Два флага к ревью (questions.md DEV-1/DEV-2):** яркие настройки (α→0 / длинный release)
    на плотном поле клиппят (бюджет амплитуды под `MASTER_GAIN=0.04` рассчитан на `1/i`);
    длинные хвосты режутся при частой смене топологии (лимит 2 банков). `MASTER_GAIN` и
    лимитер НЕ трогал (auto-gain отвергнут в decisions.md) — развилки вынесены Researcher.
  - **Ожидает прослушивания Пользователя + ревью Researcher.**

- **Громкость пре-клип + level/clip-метр (2026-06-16)** — операционное решение DEV-1 по
  выбору Пользователя: вместо угадывания headroom — ручной контроль уровня.
  - Подтверждена причина возврата кликов записью живой сессии (`CASYNTH_RECORD`): WAV
    выхода движка КЛИППИТ (int16-пик 32767, 1907 сэмплов 0.055% с 13.45 с), а
    `underruns=0`, кадров >90мс нет → НЕ старвация очереди, именно DEV-1 (стак хвостов).
  - **Громкость перенесена в ПРЕ-клип**: `render_chunk_laplacian(..., gain)` применяет
    `gain=MASTER_GAIN*vol` ДО `np.clip`; в callback'е множитель убран (`vol_box` удалён).
    Теперь снижение vol реально опускает сигнал под потолок → клип убирается, тембр честный.
    Цена — vol реагирует с задержкой look-ahead (~180мс); для ручного headroom это норма.

    сэмплов за потолком; `feed_audio` агрегирует в peak-hold (`METER_DECAY=0.90`); UI
    рисует бар (vs 0 dBFS на правом крае) + «CLIP +X.XdB» красным при перегрузе.
    Проверено пробом: яркие+длинный release на full vol → peak 1.70 (+4.6dB, 2642 клипа);
    vol 0.40 → −3.4dB, 0 клипа. Регресс `_render_probe.py` обновлён под новые сигнатуры
    (penta baseline сохранён: peak 15670/32767, 0 клипа).
  - `MASTER_GAIN` и форма спектра НЕ тронуты — финальное дизайн-решение DEV-1
    (нормировка/gain как окончательный механизм) по-прежнему за Researcher.

- **DEV-2 ИСПРАВЛЕН: клики при большом release устранены пулом tail-слотов (2026-06-16).**
  Клики при метре −15 dB (не клиппинг) — render-content артефакт. Подтверждён НА СНЕПШОТЕ
  Пользователя (не угадан): причина — старая схема 2 банков слала `amp_cur[bslot]=0` ещё
  звучащему хвосту при переиспользовании → ступенька на шве. Контрольный эксперимент:
  укорачивание release на снепшоте роняло резкие транзиенты 465→38.
  - **Фикс:** `SlotPool` переписан со схемы front/back на **active + пул tail-слотов**.
    У каждой voice×mode один active-слот; при смене частоты старая мода БЕСШОВНО
    переносится в свободный tail-слот (копируются freq/phase/amp/pan → синус продолжается
    без разрыва) и затухает там за release_chunks; active рестартует на новой частоте с
    amp 0. Старый (tail) и новый (active) звучат параллельно → overlap без щелчка, хвосты
    любой длины наслаиваются в пэд. Это же закрывает запрос на пэд из release-крутилки.
  - `update(voices, phase, amp_cur, pan_cur, release_chunks)` — теперь принимает phase и
    pan_cur (нужны для бесшовного переноса). При исчерпании пула крадётся самый тихий
    хвост (graceful, минимальный клик); счётчики `pool.steals/steal_amp_max` для диагностики.
  - **N_TAIL=384** (= 2×active; всего слотов 576). Замер на снепшоте: 192 пула не хватало
    (109 краж громких хвостов amp~0.61 → остаточный клик 1065); 384 → 0 краж, граничный
    |d2| max 1065→94.5 (= пол чистого короткого релиза), рендер +~5% (незначителен). >384
    улучшения не даёт. Плотные многообъектные поля при макс. release могут пул исчерпать —
    тогда graceful-steal (отмечено в коде).
  - **Верификация на снепшоте `_session_20260616_130029` (репро Пользователя — галактика Кока):**
    клиппинг 0; граничный |d2| max 4375→94.5, mean 138→8.1; топ-транзиены ушли со швов

## Open research — профилирование маппингов (параллельная ветка, не блокирует прототип)
Стратегия: **не выбрасывать тупиковые ветки**, набрать зоопарк приёмов побольше и
собрать **профиль/характер каждого** на разных режимах автомата → карта «какой приём
для чего годится». Рамка и шаблон профиля + текущие профили 5 приёмов — в
`memory/decisions.md` (раздел «Концепция профилирования маппингов»).
- Ближайшая измерительная задача: прогнать законсервированные приёмы (FFT/Walsh/Granulo)
  на **крупных/плотных полях** — проверить гипотезу «шумный на малом → структурный на
  большом». Формат стенда под крупные режимы не заточен (отдельная задача — surface при
  необходимости).
- Разные приёмы могут управлять РАЗНЫМИ параметрами синта (тембр/огибающая/модуляция/
  пространство) — карта это и должна развести.
