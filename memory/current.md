## Implemented (снапшот — детали в log/ и archive/)

**Прототипы и ядро**
- **`gol_synth.py`** — мульти-движковый прототип (наследник `gol_life_synth_laplacian.py`):
  тот же живой движок/UI, но приём маппинга выбирается на лету **рядом вкладок** в тулбаре
  из общего реестра `casynth_core.ENGINES` (5 приёмов со стенда: FFT/Walsh/Random/Laplace/
  Granulo). Панель крутилок **динамическая** — показывает атрибуты выбранного движка
  (Laplace: part/spread/alpha/shape/harm/full; гармонические: только part). Значения
  атрибутов **помнятся по каждому движку** (`state['engine_params']`) → возврат к движку =
  тот же звук. Маркер `fullshape` (full=1 по дефолту реестра) ветвит вызывающий код
  (`gol_synth.py:374`): `sub` целиком в Лапласиан vs `extract(sub,8)` для гармонических.
  **ADSR-огибающая** (synth-wide, отдельный UI-блок «ENV» под движковыми ручками):
  Attack/Decay/Sustain/Release привязаны к ЖИЗНИ моды (нет per-note on/off — «нота» =
  рождение/смена/смерть моды на топ-событии). Огибающая = нормированный 0..1 множитель на
  амплитуду моды из формы, тикает раз на чанк в `SlotPool` (`_env_phase`/`_env_level`/
  `_advance_env`). A: 0→пик на рождении; D: пик→sustain-уровень; S(0..1): держится всю жизнь
  моды; R: спад→0 на смерти/смене (прежняя release-механика tail-пула без изменений). Дефолты
  (A→1 чанк, D=0, **S=1**, R=50ms) = `level≡1` → `amp_tgt=amp_new` бит-в-бит как раньше.
  Щипок = S=0; пэд = S=1. `render_chunk` не тронут.
  **Лог сессии (read+write):** `CASYNTH_RECORD=1` пишет пер-кадровый `replay_controls` —
  снапшот ВСЕХ контролов (engine + note/vol/A/D/S/R + полный `engine_params`, авто-захват
  любой ручки). Чтение: `python gol_synth.py replay <ts>` (`replay_session`) — офлайн-рендер
  через те же analyse/SlotPool/render + sample-fidelity vs записанного WAV. Конвейер «запись→
  реплей→валидация фикса» замкнут. Правило: новый контрол = сразу весь путь read+write лога.
  Аудио-движок (SlotPool/глайд амплитуд): у гармонических движков частоты слотов постоянны →
  кросс-фейд хвостов не активируется; у Laplace — хвосты как прежде. Старт = laplacian с
  дефолтами реестра (part=12/spread=0/alpha=1/shape=0/harm=0, **fullshape=ON**): на формах
  ≤8×8 спектр = как у 8×8-лапласиан-прототипа, на крупных — граф по всей форме (отличается).
- `casynth_core.py` теперь содержит реестр `ENGINES` / `ENGINE_BY_ID` (id, label, map-fn,
  param-specs) — чистые данные, без pygame; единый источник для селектора движков.
- **`gol_life_synth_laplacian.py`** — предшественник (одно-движковый Laplace), оставлен как есть: каждый связный объект → голос,
  спектр = граф-Лапласиан формы (√λ → частоты мод), негармонический «металлический» тон.
  sounddevice callback-стрим (gapless, развязан с 60 fps). Живые крутилки тембра
  **partials/spread/alpha/release** (экранные слайдеры, меняются на лету; partials =
  число партиалов на объект, 1–20, дефолт 12 — твик из laplacian_explainer.html,
  пишется в лог сессии). `MAX_MODES_PER_OBJ`=20 — потолок/ёмкость слотов. Аудио-движок:
  active-слот + пул tail-слотов с бесшовным переносом хвоста (overlap-кросс-фейд без
  щелчка, хвосты наслаиваются в пэд). Громкость пре-клип + level/clip-метр.
- `gol_life_synth.py` — **legacy v1**: size→harmonic, pygame.mixer.
- `casynth_core.py` — общая либа алгоритмов маппинга форма→(freqs,amps): пять `map_*`
  (fft2d/walsh/random/laplacian/granulo), numpy/scipy без pygame.
  `map_laplacian(patch, f0, n, spread=0, alpha=1, shape=0, harm=0, fullshape=False)`;
  низшая взятая мода всегда = f0; дефолты воспроизводят исторический выход бит-в-бит.
  - **shape** [0,1]: blend между rolloff `1/i^α` (shape=0, дефолт) и проекцией краевого
    возбуждения `e_i=deg_i` на собств. векторы (shape=1 → амплитуды зависят от формы).
    Частоты не затрагиваются. Принято Researcher 2026-06-20; принято на слух 2026-06-22.
  - **harm** [0,1]: гармоническое квантование — притягивает частотные отношения мод
    `r_k=freq_k/f0` к ближайшему целому (`(1−harm)·r + harm·round(r)`), ось металл→хор→
    гармоника. Трогает ТОЛЬКО частоты; форма выбирает занятые гармоники (round-к-ближайшей,
    не 1..n). Принято Researcher (code-review) + на слух 2026-06-22.
  - **fullshape** (bool): при True вход `patch` = полный bbox объекта (не 8×8 окно) → граф
    строится по ВСЕМ клеткам; потолок `MAX_LAPLACIAN_NODES=256` с решёточной децимацией
    (`patch[::step,::step]`). Дефолт ядра False (бит-в-бит); реестровый Laplacian = True.
    Реализован 2026-06-22 (дизайн принят Researcher); прослушивание «центр vs вся форма» —
    в очереди.

## Known issues
- Segmentation не wrap-aware: объект на шве тора временно распадается на два голоса.
- Нет трекинга идентичности объектов между поколениями (нет birth/death-событий, коллизий).
- Заведено ~3 из ~8 спроектированных перцептивных осей — диапазон тембра узок.
- Клиппинг при плотном поле + яркая настройка — ручной headroom (метр+громкость, DEV-1).
  Researcher (2026-06-17): для фазы прослушивания этого достаточно; в музыкальной рабочей
  зоне (≈3–4 моды) энергия мала. Авто-нормировку уровня вернуть, только если плотные яркие
  сцены станут рабочим режимом.
