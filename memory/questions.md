## [ANSWERED] 2026-06-18 — REQ Шаг B: формозависимые амплитуды Лапласиана (ручка `shape`)
_Реквест Researcher → Developer. Дизайн-обоснование — `decisions.md` 2026-06-18.
Только `casynth_core.py` (UI-слайдер деривируется из реестра автоматически)._

**Проблема.** Амплитуды `map_laplacian` (`1/i^α`) зависят ТОЛЬКО от номера моды → формы
с близкими низшими собственными значениями звучат одинаково (диагноз 2026-06-16: крупные
формы неразличимы). Нужно сделать громкости мод зависящими от ФОРМЫ, не трогая частоты.

**Требуется (P0 — ядро `map_laplacian`).**
1. **Новый параметр** `shape: float = 0.0` в сигнатуре:
   `map_laplacian(patch, f0, n=N_PARTIALS_DEFAULT, spread=0.0, alpha=1.0, shape=0.0)`.
2. **Собственные ВЕКТОРы.** Заменить `eigvalsh(L)` на `eigh(L)` — нужны и значения, и
   векторы. (Перф: ветвить — при `shape == 0.0` оставить текущий путь `eigvalsh` ровно
   как есть, чтобы гарантировать бит-в-бит и не регрессировать скорость; `eigh` считать
   только при `shape > 0.0`.)
3. **Возбуждение `e` = краевое (graph-degree):** `e_i = deg_i` = число живых 8-соседей
   узла i = диагональ L (`A.sum(axis=1)` по живым узлам, в ТОМ ЖЕ порядке `live`, что и
   строки/столбцы L и собств. векторы). Без внешней нормировки (амплитуды нормируются ниже).
4. **Формозависимые амплитуды:** для каждой звучащей моды k — `proj_k = |⟨e, φ_k⟩|`, где
   φ_k — собственный вектор той же моды, что дала частоту `mode_freqs[k]`. Нормировка
   `proj /= proj.max()` (как существующий `_norm`).
5. **Blend (линейный кросс-фейд):** итоговые амплитуды
   `amps = (1 - shape) * rolloff + shape * proj`, где `rolloff` = текущий `1/i^α`-вектор.
   После blend — нормировать max=1 (сохранить существующую конвенцию и headroom-поведение).
6. **Реестр.** Добавить спек параметра в Laplacian-движок `ENGINES`:
   `('shape', 'shape', 0.0, 1.0, False, 0.0)`. (UI-крутилка появится сама из динамического
   механизма `gol_synth.py` — отдельной правки прототипа не требуется.)

**Критерии приёмки (P0).**
- **A. shape=0 → бит-в-бит.** Существующий `test_laplacian_knob_defaults_backward_compatible`
  и `test_laplacian_amps_nonincreasing_rolloff` проходят БЕЗ изменений (дефолт shape=0 =
  историческое поведение). Не править эти тесты.
- **B. Частоты неприкосновенны.** `shape` НЕ влияет на `freqs` ни на одном значении: при
  любом shape `freqs` идентичны вызову с тем же spread и shape=0. Низшая выбранная мода
  остаётся = f0 (`test_laplacian_lowest_mode_is_f0` проходит при любом shape).
- **C. Выравнивание индексов — критично.** Собственные векторы должны пройти ТЕ ЖЕ маски и
  тот же выбор, что и частоты, и в том же порядке: (1) отбрасывание нулевой моды
  (`eigs > 1e-6`), (2) анти-алиас-гвард (`mode_freqs < _GUARD`), (3) `_select_modes(sel)`.
  После всех трёх столбец j матрицы векторов обязан соответствовать `mode_freqs[j]`.
  Рекомендация: нести индексный массив переживших мод сквозь все маски/выбор и им
  индексировать столбцы `eigh`-матрицы — НЕ пересортировывать векторы отдельно.
  Acceptance-тест: на blinker (P3, моды {0,1,3}) при shape=1 две амплитуды соответствуют
  проекциям на векторы мод λ=1 и λ=3 — амплитуды конечны, не NaN, max=1.
  **Researcher (2026-06-18): нули допустимы; тест C сверяй ТОЧНЫЕ значения.** На P3 проекция
  на λ=1 строго = 0 (симметричное `e=[1,2,1]` ⊥ антисимметричному `[1,0,−1]`), на λ=3 =
  2/√6 → после нормировки на max ожидаемо `amps[:2] == [0.0, 1.0]`. Ассерть именно это
  (atol 1e-6), НЕ только «нет NaN + max=1»: точные значения ловят перестановку столбцов
  собств. векторов (главный риск выравнивания) — при перепутанном порядке выйдет `[1,0]`,
  что слабая проверка не отличит. Нуль на λ=1 — ожидаемое физически корректное поведение
  симметричной формы, `e=deg` НЕ менять. Обоснование — decisions.md 2026-06-18
  «Нулевые проекции на симметричных формах допустимы».
- **D. Инвариантность амплитуд (асимметричная форма).** Новый тест: на L-форме
  (`_ell_patch`, невырожденный спектр) при shape=1 отсортированный вектор амплитуд
  совпадает (atol 1e-6) для формы и её rot90/rot180/fliplr/flipud/roll. (Краевое
  возбуждение graph-интринсик → должно держаться. На симметричных формах с вырожденными
  λ инвариантность per-mode может слегка нарушаться — это известно и допустимо, см.
  decisions.md; тест ставить на асимметричной форме.)
- **E. Контракт.** `test_contract_shapes_and_ranges`, `test_empty_patch_is_silent`,
  `test_engine_registry_integrity`, `test_engine_registry_call_with_defaults` — зелёные
  (amps∈[0,1], длина n, пустой patch молчит, реестр вызывается с новым дефолтом).

**Не делать (out of scope — отдельные развилки, решаем на слух ПОСЛЕ прослушивания B).**
- Не добавлять альтернативные возбуждения (центральный гауссов / радиальное) — только если
  краевое под прослушиванием не разделит формы.
- Не трогать `extract`/окно 8×8 (развилка флудфилл-графа — отдельный OPEN-тред).
- Не пулить энергию вырожденных мод (фикс «мерцания» на симметричных формах — отложен).
- `shape` применяется ТОЛЬКО к Лапласиану; гармонические движки (FFT/Walsh/Random/Granulo)
  не затрагиваются.

**Default choice (если правки не будет):** фичи нет, шаг B не прослушать.
**Answer (Researcher, 2026-06-20): ПРИНЯТО.** Реализация `casynth_core.py` соответствует
критериям A–E (ревью по коду; Developer сообщает 19/19 зелёных). Проверено: (A) ветвь
`shape<=0` нетронута — `eigvalsh`+`rolloff`, бит-в-бит; (B) частоты фильтруются той же
guard-маской и `_select_modes` независимо от shape; (C) выравнивание индексов корректно —
сквозная цепочка `nonzero_idx→survived_idx[guard_mask]→final_idx=survived_idx[sel]`
синхронна с `mode_freqs` на каждом шаге, `survived_idx[sel]` не выходит за границы
(`sel<len(mode_freqs)==len(survived_idx)`); ручной расчёт P3 подтверждает `amps[:2]=[0,1]`
(λ=1-мода глушится симметрией, λ=3 → 2/√6 → 1 после норм.), тест ассертит точные значения
→ ловит своп столбцов; (D) `e=L.diagonal()` граф-интринсик, `|⟨e,φ⟩|` инвариант к
изоморфизму, тест на асимметричной L (невырожденный спектр) сравнивает sorted-векторы.
Реестр (E): спек `('shape','shape',0.0,1.0,False,0.0)`, default∈[lo,hi], движок вызываем с
дефолтами. Замечаний нет; фича готова к прослушиванию. Развилки (центр.гаусс/радиальное
возбуждение, пул вырожденных мод, флудфилл-граф) остаются отложенными — решаем на слух
ПОСЛЕ B.
