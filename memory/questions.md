## [OPEN] 2026-06-21 — REQ GoL-вокодер (T3): offline-прототип креативного ресинтеза
_Реквест Researcher → Developer. Дизайн — `decisions.md` 2026-06-21 «GoL-вокодер (T3)».
Новый offline-скрипт (напр. `gol_vocoder.py`), НЕ живой синт._

**Цель.** Креативный ресинтез: поиском найти бинарное GoL-поле, чей спектр прямого маппинга
приближает спектр целевого звука. Результат — эффект «цель сквозь GoL-металл», не точная копия.
Точная инверсия невозможна (коспектральность) — приближение есть характер.

**Требуется.**
1. **Переиспользовать `casynth_core.map_laplacian`** напрямую (это Python — НЕ дублировать
   движок, как в explainer; импортировать настоящий → паритет звука).
2. **Анализ цели.** Вход — WAV (моно-кадр стационарного участка). FFT → top-K пиков
   (частота+магнитуда). Отпечаток = **Bark-огибающая** (магнитуды по Bark-/лог-частотным
   бинам), якорь по низшему заметному партиалу = f0 (как `sel[0]` в ядре оставляет низшую
   моду = f0).
3. **Поиск — simulated annealing.** Состояние = бинарное поле фикс-окна (старт 8×8, размер —
   конфиг). Ход = флип случайной клетки (или малого блока). Eval кандидата: прогнать поле
   через `map_laplacian` (n/spread/alpha/shape из конфига) → (freqs,amps) → та же Bark-огибающая
   → дистанция `1 − cosine` к огибающей цели. Приём по Метрополису с температурным расписанием +
   **случайные рестарты** (держать глобально лучшее). По умолчанию **`shape>0`** (≈0.5–1), чтобы
   в матч входили и амплитуды, и n высокое (плотный спектр под цель).
4. **Выход.** Лучшее поле → (а) рендер WAV тем же движком; (б) сохранить как загружаемый
   паттерн (формат `patterns.py` / grid, чтобы можно было кинуть в `gol_synth.py` и при желании
   запустить эволюцию); (в) A/B-пара «цель vs результат» для прослушивания; (г) печать score+поля.
5. **Лог** (правило памяти): записать цель, найденное поле, score и ВСЕ параметры поиска/рендера
   (window, n, spread, alpha, shape, seed, расписание SA) — прогон воспроизводим.

**Критерии приёмки.**
- **Bring-up self-test (главный):** подать СОБСТВЕННЫЙ спектр известного поля (напр. blinker или
  случайного 8×8) как цель → SA восстанавливает поле с Bark-дистанцией ниже явного порога
  (в идеале → исходное поле или коспектральное ему). Доказывает, что петля анализ→поиск→рендер
  замкнута и метрика осмысленна.
- На синтетической цели (напр. гармонический стек) поиск **монотонно снижает** дистанцию
  относительно случайного старта; лучшее поле звучит как «цель сквозь металл», без NaN/ошибок.
- Рендер идёт **только** через `casynth_core` (никакого второго движка); смена `shape`/n/spread
  меняет результат согласованно с тем, как звучал бы синт на том же поле.
- Скрипт детерминирован при фиксированном seed; лог достаточен для повторного прогона.

**Default choice (если правки не будет):** прототипа нет — вокодер не послушать, ветка T3 закрыта.

## [OPEN] 2026-06-21 — REQ explainer: ручной mode-toggle + слайдер shape + убрать matrix-блок
_Реквест Researcher → Developer. Дизайн — `decisions.md` 2026-06-21. Файл:
`laplacian_explainer.html` (discover-инструмент/обучалка, НЕ синт; JS дублирует map_laplacian)._

**Цель.** Дать Пользователю наработать на слух карту «какие моды Лапласиана звучат чисто/грязно
на разных нотах». Диагноз: грязь = roughness от скучивания мод в критполосах. До автоматизации
отбора мод — ручной выбор. Инструмент должен звучать как синт (с shape), иначе интуиция не
перенесётся.

**Требуется.**
1. **Ручная маска мод (главное).** Каждый столбик-мода в спектре кликабелен → вкл/выкл. Звук и
   все панели (конвейер, таблица, спектр) пересчитываются по выбранной маске мгновенно.
   - Кандидаты — звучащие моды формы (`modeFreqs` после guard), показывать с потолком ~20.
   - Стартовая маска = текущее поведение (первые `n` включены при spread=0) → старт совпадает с
     синтом.
   - **Нормировка амплитуд (max=1) и shape-проекции — ТОЛЬКО по включённым модам** (паритет с
     тем, как синт нормирует выбранное подмножество).
   - **Низшая ВКЛЮЧЁННАЯ мода = f0** (как `sel[0]` в ядре): выключение самой низшей делает f0
     следующей включённой, спектр перемасштабируется. Привязка к ноте держится на любой маске.
   - `spread` оставить как генератор стартовой маски (двинул → галочки расставились децимацией,
     дальше правишь руками) ЛИБО спрятать в этом режиме — на твоё усмотрение; главный механизм =
     ручные галочки.
2. **Слайдер `shape ∈ [0,1]`** (паритет со звуком синта). Как в `casynth_core.map_laplacian`:
   `amps = (1-shape)*rolloff + shape*proj`, `proj_k = |⟨diag(L), φ_k⟩|` по включённым модам;
   нормировка max=1 после blend. Собств. векторы уже есть (`jacobiEigen` → `res.vectors`). Дефолт 0.
3. **Убрать интерактивный блок «Матрицы формы»** — canvas `#mat` + радио A/D/L + hover-логика и
   связанный JS (`drawMatrix`, mousemove/mouseleave-листенеры, `currentMatType`, `matCells`,
   `DEFAULT_MATHINT`, вызов в `redrawAll`). **Статичную инфографику Части 1 (шаг 3, blinker
   L=D−A) НЕ трогать** — это обучение, а не контрол.

**Критерии приёмки.**
- Клик по столбику-моде вкл/выкл; звук/спектр/таблица/конвейер пересчитываются мгновенно.
- `shape=0` → амплитуды бит-в-бит как сейчас (rolloff `1/i^α`). `shape>0` двигает амплитуды;
  вектор берётся для ТОЙ ЖЕ моды, что дала частоту (выравнивание индексов как в ядре).
- Нормировка и проекции считаются по ВКЛЮЧЁННЫМ модам; низшая включённая нормируется к f0.
- Алгоритм частот/масштаба (`computeLaplacian` pipeline) не тронут — `shape`/маска влияют только
  на выбор мод и амплитуды, не на √λ→Гц.
- Интерактивный matrix-блок удалён; консоль без ошибок; остальные панели работают.

**Default choice (если правки не будет):** инструмента нет — алгоритм автоотбора выбираем
вслепую, без слуховых данных.

## [OPEN] 2026-06-21 — REQ explainer: секция «Краевое возбуждение → формозависимые амплитуды» (4 панели)
_Реквест Researcher → Developer. Дизайн — `decisions.md` 2026-06-18 (краевое возбуждение `e=deg`) +
эта сессия. Файл: `laplacian_explainer.html` (обучалка; JS дублирует map_laplacian — известный
trade-off). Новая обучающая СЕКЦИЯ; переиспользует уже считающиеся `jacobiEigen`→`res.vectors` и
гистограмму мод. НЕ конфликтует с REQ mode-toggle (тот про ВЫБОР мод, этот про ОБЪЯСНЕНИЕ громкостей)
— встают в один файл, можно делать пакетом._

**Цель.** Объяснить на глаз+слух, что такое `e` (краевое возбуждение, `e_i = deg_i` = число живых
8-соседей = диагональ `L`) и как из него рождаются формозависимые амплитуды (`shape`). Стержень:
форма «звучит своей границей» (однородная внутренность `deg=8` нема — ⟂ звучащим модам; звучит
разница `deg` на кромке).

**4 панели (сквозной герой — вытянутая «струна» 2×6; + контрастные пресеты).**
1. **A — «Форма звучит границей».** Клетки раскрашены по `e=deg` (тепло). Переключатель
   `deg ↔ дефицит (8−deg)`: в «дефиците» внутренность гаснет в 0, светится только кромка.
2. **B — «У моды есть ФОРМА».** Клик по столбику-моде → форма раскрашена по `φ_k`
   (red/blue/white = +/−/узел = фигура Хладни); мода играет solo. На «струне» моды = гармоники
   (ниже = плавный градиент, выше = частые чередования).
3. **C — «Громкость = перекрытие удара с модой».** Рядом `e` и `φ_k`; ПОКЛЕТОЧНОЕ произведение
   `e_i·φ_k(i)` раскраской (совпали по знаку = вклад, противоположны = гасят); сумма = `⟨e,φ_k⟩`
   → громкость (число). Слайдер `shape 0→1`: столбик моды переезжает с `1/i^α` на проекцию.
4. **D — «Почему симметрия глушит моду».** Пресет blinker: `e=[1,2,1]` симметричен,
   антисимметричная `φ` (λ=1) → произведение `+0−` суммируется в 0 → столбик = 0 (показать и
   ненормированную сумму=0, и нормированный 0). Пресет «blinker+1» (сломана симметрия) → мода
   оживает. «Симметричная пластина не звенит антисимметричной модой».

**Наглядные/«наслушные» пресеты (кнопки «загрузить форму»; все панели пересчитываются).**
- **«Компактный блок» (3×4)** и **«Протяжённая змейка» РАВНОЙ массы (12 клеток)** — пара для A:
  одинаковая площадь, разная граница → разный паттерн возбуждения. (Как звучит — Пользователь
  оценивает на слух; перцептивное предсказание в критерий НЕ зашивать.)
- **«Струна» (2×6 / линия)** — сквозной герой B+C: моды-гармоники с явными узлами; краевой удар на
  концах → пучности низшей моды (громко) vs узлы иных (тихо).
- **«Blinker»** и **«Blinker+1»** — пара для D.

**Критерии приёмки.**
- Панели пересчитываются по выбранному пресету и выбранной моде; раскраски корректны: `deg`/«дефицит»,
  знак `φ_k` (узлы≈0), знак поклеточного произведения.
- `shape`-проекция в C — БИТ-в-бит как в ядре/mode-toggle (`e=diag(L)`, выравнивание векторов по той
  же моде, что дала частоту, нормировка max=1 по включённым). `shape=0` → столбики = `1/i^α`.
- **Математически проверяемо:** blinker глушит антисимметричную моду (ненорм. сумма `⟨e,φ⟩=0`,
  нормированный столбик = 0); «blinker+1» — та мода ненулевая. (Это и есть тест выравнивания.)
- `e` у «компактного» и «протяжённого» пресетов ВИДИМО различны на форме (паттерн разный);
  слуховой контраст — гипотеза для Пользователя, НЕ критерий кода.
- Solo-проигрыш моды на её частоте; статичную инфографику Части 1 (blinker `L=D−A`) НЕ трогать;
  консоль без ошибок; mode-toggle и прочие панели работают.

**Не делать (out of scope).**
- Не привязывать к ЖИВОМУ полю/динамике (`e(t)`, переливы во времени — отдельная развилка
  «возбуждение как модуляция», в работу не взята).
- Не вводить альтернативные возбуждения (центр.гаусс/радиальное) — секция объясняет ТЕКУЩЕЕ `e=deg`.
- Не менять алгоритм частот/масштаба (`computeLaplacian`) — панели влияют только на показ.

**Default choice (если правки не будет):** `e` остаётся «таинственным» — механизм `shape` непонятен,
дизайн возбуждений ведём вслепую.

### REQ Флудфилл-граф для Лапласиана (full-shape mode)

**Ключевая находка (она снимает половину вопросов).** Лапласов путь в `map_laplacian`
читает форму ТОЛЬКО через `np.argwhere(patch > 0)` + 8-окрестность по координатам
(строки 190–204). Он НИКОГДА не предполагает квадрат, 8×8 или регулярную решётку — граф
строится по живым клеткам как есть. То есть ядро УЖЕ умеет работать на произвольной маске;
единственный ограничитель — что вызывающий код кормит `extract(sub, 8)` (кроп до 8×8),
а не саму маску `sub`. И `sub` уже построен вызывающим кодом: в `gol_synth.py:371-372` и
`gol_life_synth_laplacian.py` каждый объект сегментируется (`ndimage.label`), и из него
делается ТОЧНЫЙ bbox-битмап `sub` (только клетки этого объекта). `extract(sub,8)` его потом
обрезает. **Флудфилл-граф = просто перестать обрезать: подать `sub` в Лапласиан напрямую.**

**Ответы на 7 вопросов.**

**1. Контракт `map_laplacian`.** Сигнатуру НЕ ломаем, добавляем флаг последним аргументом:
   `map_laplacian(patch, f0, n=..., spread=0.0, alpha=1.0, shape=0.0, harm=0.0, fullshape=False)`.
   - При `fullshape=False` (дефолт) — поведение бит-в-бит как сейчас (вход — 8×8 патч).
   - При `fullshape=True` — `patch` трактуется как **произвольный бинарный bbox-битмап одного
     объекта** (то, что вызывающий код кладёт в `sub`): любая высота×ширина, фон=0, объект=1.
   Внутри ядра НИКАКОЙ ветвёжки по `fullshape` НЕ нужно для самого построения графа — он уже
   агностичен к размеру. Флаг нужен только как явный СЕМАНТИЧЕСКИЙ переключатель «вход — патч
   vs полная маска» (чтобы реестр/вызов знали, что слать) и как точка крепления guard'а по
   числу узлов (см. п.4). Альтернатива «отдельная функция `map_laplacian_full`» отвергнута:
   тело идентично на 99%, дублирование = риск дивергенции (ровно то, против чего заведён
   `casynth_core`); один флаг чище.

**2. Другие движки — асимметрию разрешаем НА ВЫЗОВЕ, не в контракте каждого движка.**
   FFT/Walsh/Random/Granulo остаются строго на 8×8 (им нужна регулярная решётка фикс-размера;
   `_fft_order`/`_walsh_order`/`_R_MAT` прекомпьютены под `PATCH_SIZE`). НЕ делать `fullshape`
   общим параметром всех движков. Маркер в реестре `ENGINES`: добавить булев ключ движка
   `fullshape=True` ТОЛЬКО Лапласиану (остальным — отсутствует/False). Вызывающий код ветвит:
   ```
   spec = ENGINE_BY_ID[engine_id]
   shape_in = sub if spec.get('fullshape') else extract(sub, PATCH_SIZE)
   freqs, amps = fn(shape_in, f0, **kwargs)   # kwargs включает fullshape=spec.get('fullshape', False)
   ```
   `sub` у вызывающего уже есть — лишних вычислений ноль; `extract` зовётся лениво только для
   гармонических. Это инженерия вызывающего кода (`gol_synth.py`) — НЕ под дизайн-ревью; здесь
   фиксирую лишь контракт, который ядро обязано поддержать (флаг + маркер реестра).

**3. Границу объекта извлекает ВЫЗЫВАЮЩИЙ код — он это УЖЕ делает.** Маска `sub`
   (`gol_synth.py:369-372`) — тот самый флудфилл-результат `ndimage.label` в тайтовом bbox.
   `casynth_core` НЕ трогает `ndimage.label` и НЕ занимается сегментацией — это разделение
   ответственности соблюдается и сейчас (ядро получает уже изолированную форму). С флудфиллом
   ничего не меняется: вызывающий просто перестаёт кропать `sub` перед подачей в Лапласиан.

**4. Крупные объекты — потолок узлов ОБЯЗАТЕЛЕН (`eigh` это O(V³)).** В full-shape добавить
   константу `MAX_LAPLACIAN_NODES` (старт **256** — детерминированный потолок, ниже tunables).
   Если число живых клеток объекта > потолка — **детерминированный даунсэмпл маски** до
   укладки в потолок ПЕРЕД построением графа: децимация решётки целым шагом
   `step = ceil(sqrt(V / MAX))` (`mask[::step, ::step]`), затем при необходимости добор —
   но проще и достаточно фиксированной децимации по шагу; шаг детерминирован от `V` →
   воспроизводимо. ВАЖНО: даунсэмпл должен оставаться инвариантным к позиции (децимация от
   bbox-origin маски, не от позиции в поле — `sub` уже origin-relative, инвариантность держится).
   Перф-замечание: при `shape=0`/`harm=0` и малом `n` мы всё равно считаем ВЕСЬ спектр
   (`eigvalsh`), берём низшие — потолок 256 держит `eigvalsh` в ~единицы мс, приемлемо для
   объектов Life (они редко >256 клеток). Шумный верх плотного спектра — НЕ проблема в рабочей
   зоне (берём мало низших/разнесённых мод, decisions.md 2026-06-17); потолок здесь про
   стоимость `eigh`, не про тембр.

**5. Совместимость `spread/alpha/shape/harm` — смысл СОХРАНЯЕТСЯ, оси работают как есть.**
   Все четыре оперируют ПОСЛЕ построения спектра, над `mode_freqs`/собств.векторами/амплитудами —
   они агностичны к тому, граф 8×8 или флудфилл. Нюансы (НЕ требуют кода, для прослушивания):
   - `spread`: на крупном графе спектр длиннее → `spread=1` дотягивается ВЫШЕ (больше «материала»
     наверху). Это и есть желаемый эффект флудфилла — периферия формы теперь слышна. Низшая
     выбранная = f0 держится (`sel[0]=0`).
   - `shape`: краевое возбуждение `e=deg` теперь видит ВСЮ границу объекта, а не кромку 8×8-кропа
     → ровно цель развилки (крупные формы различаются периферией). `|⟨e,φ_k⟩|` инвариантно к
     изоморфизму графа на маске любого размера.
   - `harm`: квантование отношений `r_k` — поэлементная ф-я, безразлична к размеру.
   - `alpha`: rolloff `1/i^α` по номеру выбранной моды — без изменений.
   НИЧЕГО из них не меняет смысл; full-shape лишь даёт им более богатый спектр на вход.

**6. Обратная совместимость — `extract` НЕ выбрасывать, `fullshape=False` НЕ удалять.**
   Причины: (а) гармонические движки ОБЯЗАНЫ остаться на `extract`/8×8 (п.2) — функция нужна;
   (б) дефолт `fullshape=False` = бит-в-бит исторический Лапласиан → все baseline-тесты держатся
   без правок (load-bearing для процедуры ревью); (в) 8×8-режим — валидная A/B-точка для
   прослушивания «центр vs вся форма» (Пользователь сравнивает на слух — без отката сравнить
   нельзя). Переключатель: новый Laplacian-движок с `fullshape=True` ЛИБО держать оба как
   варианты. Рекомендация: **сделать `fullshape=True` дефолтом РЕЕСТРОВОГО Laplacian-движка**
   (это то поведение, которое хотим слышать), но ядро дефолтит `fullshape=False` (бит-в-бит для
   тестов). То есть: ядро консервативно, реестр прогрессивен. Если на слух full-shape окажется
   хуже на каких-то режимах — откат = снять маркер в реестре, код ядра не трогаем.

**7. Тесты — инварианты те же, но на ПРОИЗВОЛЬНОЙ маске (не 8×8).**

**Критерии приёмки.**
- **A. fullshape=False → бит-в-бит.** ВСЕ существующие тесты Лапласиана
  (`..._backward_compatible`, `..._lowest_mode_is_f0`, `..._uses_sqrt_lambda`, shape-тесты,
  harm-тесты, `..._invariant_to_rotation_reflection_translation`) проходят БЕЗ изменений.
  Не править их. Дефолт `fullshape=False` = исторический путь.
- **B. fullshape на ≤8×8 форме = тот же спектр, что 8×8-путь.** Для формы, целиком влезающей
  в 8×8 (напр. `_ell_patch`, `_blinker_patch`), `map_laplacian(sub, f0, fullshape=True)` даёт
  ТЕ ЖЕ sorted-freqs и sorted-amps (atol 1e-6), что `map_laplacian(extract(sub,8), f0)` —
  где `sub` = тайтовый bbox той же формы. Доказывает, что full-режим — не новый приём, а
  снятие кропа: на формах, которые кроп НЕ резал, результат идентичен. (Тонкость: центрирование
  в 8×8 vs тайтовый bbox дают РАЗНЫЕ массивы, но ОДИН граф живых клеток → один спектр. Если
  тест падает — значит ядро где-то зависит от размера массива/фона; это баг, лови его.)
- **C. Крупная форма звучит ПЕРИФЕРИЕЙ (главный тест развилки).** Форма >8×8 (напр.
  L-образная 12×12 с длинными плечами): `map_laplacian(..., fullshape=True)` даёт спектр,
  ОТЛИЧНЫЙ от 8×8-кропа той же формы (sorted-freqs различаются заметно — не atol-равны).
  Доказывает, что периферия теперь влияет на звук. (Дополнительно при shape=1: амплитуды
  full-shape ≠ амплитудам 8×8-кропа — краевое возбуждение видит всю границу.)
- **D. Инвариантность на произвольной маске.** На асимметричной форме >8×8 sorted-freqs
  (и при shape=1 sorted-amps) при `fullshape=True` совпадают (atol 1e-6) для маски и её
  rot90/rot180/fliplr/flipud. (Трансляцию НЕ тестируем массивом — `sub` уже origin-relative;
  вместо неё проверить, что добавление пустых рамок фона вокруг маски НЕ меняет спектр:
  фон=0 не даёт узлов графа.)
- **E. Привязка f0 при full-shape.** `freqs[0] == f0` (atol 1e-6) при `fullshape=True` для
  любых spread/shape/harm∈{0,0.5,1} (как сейчас — `sel[0]=0`, `nonzero_sq[0]→f0`).
- **F. Потолок узлов детерминирован и инвариантен.** Форма с V>`MAX_LAPLACIAN_NODES` живых
  клеток: (1) `eigh`/`eigvalsh` вызывается на ≤MAX узлах (не падает по памяти/времени —
  можно проверить на синтетическом плотном блоке, напр. 20×20=400>256); (2) результат
  детерминирован (два вызова на той же форме идентичны); (3) даунсэмпл инвариантен к
  rot90/flip большой формы (sorted-freqs совпадают atol — допускается чуть бо́льший atol,
  т.к. децимация может слегка нарушить точную симметрию; если строгий 1e-6 не держится,
  ослабить до 1e-3 И зафиксировать это в тесте как известный эффект децимации).
- **G. Контракт/реестр.** `test_contract_shapes_and_ranges`, `test_empty_patch_is_silent`,
  `test_engine_registry_integrity`, `..._call_with_defaults` — зелёные. Пустая/одноклеточная
  маска при `fullshape=True` молчит (`cnt<2` → нули, как сейчас). Реестр: Laplacian имеет
  маркер `fullshape=True`; `test_engine_registry_call_with_defaults` зовёт `fn` корректно
  (передаёт `fullshape` если движок его объявляет — иначе дефолт ядра `False`).

**Не делать (out of scope — отдельные развилки/инженерия).**
- НЕ менять контракт гармонических движков (FFT/Walsh/Random/Granulo остаются 8×8).
- НЕ переносить ветвёжку `sub vs extract` в ядро — это инженерия вызывающего кода
  (`gol_synth.py`), под дизайн-ревью не попадает; ядро лишь обязано принять флаг + маркер.
- НЕ менять алгоритм даунсэмпла на «умный» (морфологический skeleton, медиана и т.п.) —
  простая решёточная децимация по шагу достаточна для держания стоимости `eigh`; умный
  ресемпл — отдельная дверь, только если на слух потолок режет характер крупных форм.
- НЕ трогать `harm`/`shape`/`spread`/`alpha` (работают как есть на любом размере графа).
- Взвешенный Лапласиан (диагональ 1/√2, decisions.md 2026-06-16) — отдельный приём, не сюда.
