# Developer Questions Queue

Developer appends questions here when a design decision is needed.
User or Researcher answers in-line.

## Entry format
```
## [OPEN] YYYY-MM-DD — [Topic]
**Question:** ...
**Context:** what is being implemented and why this matters.
**Default choice:** what was done in the absence of an answer.
**Answer:** _(to be filled)_
```

Change `[OPEN]` to `[ANSWERED]` once resolved.

## [OPEN] 2026-06-15 — Форма стенда для перебора маппингов: режим vs отдельное приложение
**Question:** Исследовательский этап (см. `decisions.md` 2026-06-15) требует быстро
перебирать разные алгоритмы маппинга `форма → вектор амплитуд` на фиксированном
зоопарке осцилляторов в окне 8×8. Как это оформить: (а) отдельный режим внутри
основного приложения, (б) отдельное приложение, переиспользующее тот же аудио-движок,
(в) что-то иное?
**Context:** Маппинг договорились делать сменным модулем (плагин: форма → амплитуды),
движок при этом один. Стенд нужен под **batch-прослушивание матрицы
`осциллятор × маппинг`**: реализуем несколько маппингов сразу (см. очередь пп. 1–5 в
`decisions.md`), прогоняем фиксированный зоопарк (блинкер, маяк, жаба, часы),
переключаемся между ячейками и сравниваем тембры на слух. Поднято Пользователем на
дизайн-сессии; решение архитектурное → за Разработчиком.
**Default choice:** _(не реализовано)_
**Answer:** _(Разработчику — на твоё усмотрение)_
