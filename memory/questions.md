## [OPEN] 2026-06-21 — REQ Гармоническое квантование частот Лапласиана (ручка `harm`)
_Реквест Researcher → Developer. Дизайн — `decisions.md` 2026-06-21 «Гармоническое квантование».
Только `casynth_core.py` (UI-слайдер деривируется из реестра автоматически, как `shape`)._

**Цель.** Непрерывная ось «инхармонично↔гармонично»: ручка `harm∈[0,1]` притягивает частоты
Лапласиана к целой гармонической решётке. 0=металл (как сейчас), середина=расстроенный хор,
1=гармоника. Форма остаётся слышна (Вариант I: round к ближайшей гармонике).

**Требуется (ядро `map_laplacian`).**
1. **Новый параметр** `harm: float = 0.0` — последним в сигнатуре:
   `map_laplacian(patch, f0, n=..., spread=0.0, alpha=1.0, shape=0.0, harm=0.0)`.
2. **Квантование частотных отношений.** После `_select_modes` (моды выбраны), для отношений
   `r_k = mode_freqs[k] / mode_freqs[sel[0]]` (низшая выбранная → r_0=1):
   `r_k(harm) = (1 − harm)·r_k + harm·round(r_k)`; итог `freqs = f0 · r_k(harm)`.
   Округление — к БЛИЖАЙШЕМУ целому (`np.rint`/`round`). НЕ последовательное присвоение 1..n.
3. **Бит-в-бит при harm=0.** Ветвить: при `harm <= 0.0` оставить текущий путь построения freqs
   ровно как есть (никакого round) — гарантия бит-в-бит и нет лишних вычислений.
4. **Только частоты.** Амплитуды (rolloff `1/i^α` + shape-проекции) НЕ трогать — harm не входит
   в их расчёт ни на каком значении.
5. **Реестр.** Добавить спек в Laplacian-движок `ENGINES`: `('harm', 'harm', 0.0, 1.0, False, 0.0)`.
   (UI-крутилка появится сама из динамического механизма `gol_synth.py`.)
6. **Лог сессии.** Убедиться, что `harm` попадает в `replay_controls` (если `engine_params`
   авто-захватывает любую ручку — подхватится; проверить, что новый контрол в логе read+write —
   правило памяти «весь путь read+write лога»).

**Критерии приёмки.**
- **A. harm=0 → бит-в-бит.** Все существующие baseline-тесты (`..._backward_compatible`,
  `..._lowest_mode_is_f0`, shape-тесты A–E) проходят БЕЗ изменений. Не править их.
- **B. Амплитуды неприкосновенны.** При любом harm `amps` бит-в-бит как при harm=0 (тот же
  spread/alpha/shape) — harm не входит в расчёт громкостей.
- **C. harm=1 → целочисленные отношения.** На асимметричной форме (`_ell_patch`, невырожд.
  спектр) при harm=1: `freqs/f0 ≈ rint(freqs/f0)` (atol 1e-6) для всех мод; низшая = f0 (отн. 1).
- **D. Привязка f0 при любом harm.** `test_laplacian_lowest_mode_is_f0` проходит при
  harm∈{0, 0.5, 1.0}: `freqs[0] == f0` (atol 1e-6). (r_0=1 → round=1 → не сдвигается.)
- **E. Линейность blend.** Для каждой моды `freqs(harm=0.5)` лежит ровно посередине между
  `freqs(harm=0)` и `freqs(harm=1)` (atol 1e-6) — ловит, что blend применён к частотам (не к λ).
- **F. Инвариантность.** На L-форме sorted-`freqs` при harm=1 совпадают (atol 1e-6) для формы и
  её rot90/rot180/fliplr/flipud/roll (квантование — поэлементная ф-я от инвариантных r_k).
- **G. Контракт/реестр.** `test_contract_shapes_and_ranges`, `test_empty_patch_is_silent`,
  `test_engine_registry_integrity`, `..._call_with_defaults` — зелёные; freqs длины n, конечны,
  >0; amps∈[0,1]; реестр вызывается с новым дефолтом harm=0.

**Не делать (out of scope — отдельные развилки, на слух ПОСЛЕ прослушивания).**
- Дедупликация коллидирующих мод у предела (две моды → одно целое): оставить как есть (два слота);
  на ровно harm=1 редко сидят, в рабочей зоне <1 они бьются = фича. Фикс слияния — отложен.
- Лог-пространство blend (полутона вместо линейного ratio) — только если притяжение на слух
  неравномерно.
- Snap-to-хроматической-шкале; выбор класса гармоник (нечётные→меандр) — отдельные фичи/двери.
- harm применяется ТОЛЬКО к Лапласиану; гармонические движки (FFT/Walsh/Random/Granulo) не трогать.

**Default choice (если правки не будет):** фичи нет, ось металл↔гармоника не прослушать.

## [OPEN] 2026-06-21 — REQ GoL-вокодер (T3): offline-прототип креативного ресинтеза
_Реквест Researcher → Developer. Дизайн — `decisions.md` 2026-06-21 «GoL-вокодер (T3)».
Новый offline-скрипт (напр. `gol_vocoder.py`), НЕ живой синт._

**Цель.** Креативный ресинтез: поиском найти бинарное GoL-поле, чей спектр прямого маппинга
приближает спектр целевого звука. Результат — эффект «цель сквозь GoL-металл», не точная копия.
Точная инверсия невозможна (коспектральность) — приближение есть характер.

**Требуется.**
1. **Переиспользовать `casynth_core.map_laplacian`** напрямую (это Python — НЕ дублировать
   движок, как в explainer; импортировать настоящий → паритет звука).
2. **Анализ цели.** Вход — WAV (моно-кадр стационарного участка). FFT → top-K пиков
   (частота+магнитуда). Отпечаток = **Bark-огибающая** (магнитуды по Bark-/лог-частотным
   бинам), якорь по низшему заметному партиалу = f0 (как `sel[0]` в ядре оставляет низшую
   моду = f0).
3. **Поиск — simulated annealing.** Состояние = бинарное поле фикс-окна (старт 8×8, размер —
   конфиг). Ход = флип случайной клетки (или малого блока). Eval кандидата: прогнать поле
   через `map_laplacian` (n/spread/alpha/shape из конфига) → (freqs,amps) → та же Bark-огибающая
   → дистанция `1 − cosine` к огибающей цели. Приём по Метрополису с температурным расписанием +
   **случайные рестарты** (держать глобально лучшее). По умолчанию **`shape>0`** (≈0.5–1), чтобы
   в матч входили и амплитуды, и n высокое (плотный спектр под цель).
4. **Выход.** Лучшее поле → (а) рендер WAV тем же движком; (б) сохранить как загружаемый
   паттерн (формат `patterns.py` / grid, чтобы можно было кинуть в `gol_synth.py` и при желании
   запустить эволюцию); (в) A/B-пара «цель vs результат» для прослушивания; (г) печать score+поля.
5. **Лог** (правило памяти): записать цель, найденное поле, score и ВСЕ параметры поиска/рендера
   (window, n, spread, alpha, shape, seed, расписание SA) — прогон воспроизводим.

**Критерии приёмки.**
- **Bring-up self-test (главный):** подать СОБСТВЕННЫЙ спектр известного поля (напр. blinker или
  случайного 8×8) как цель → SA восстанавливает поле с Bark-дистанцией ниже явного порога
  (в идеале → исходное поле или коспектральное ему). Доказывает, что петля анализ→поиск→рендер
  замкнута и метрика осмысленна.
- На синтетической цели (напр. гармонический стек) поиск **монотонно снижает** дистанцию
  относительно случайного старта; лучшее поле звучит как «цель сквозь металл», без NaN/ошибок.
- Рендер идёт **только** через `casynth_core` (никакого второго движка); смена `shape`/n/spread
  меняет результат согласованно с тем, как звучал бы синт на том же поле.
- Скрипт детерминирован при фиксированном seed; лог достаточен для повторного прогона.

**Default choice (если правки не будет):** прототипа нет — вокодер не послушать, ветка T3 закрыта.

## [OPEN] 2026-06-21 — REQ explainer: ручной mode-toggle + слайдер shape + убрать matrix-блок
_Реквест Researcher → Developer. Дизайн — `decisions.md` 2026-06-21. Файл:
`laplacian_explainer.html` (discover-инструмент/обучалка, НЕ синт; JS дублирует map_laplacian)._

**Цель.** Дать Пользователю наработать на слух карту «какие моды Лапласиана звучат чисто/грязно
на разных нотах». Диагноз: грязь = roughness от скучивания мод в критполосах. До автоматизации
отбора мод — ручной выбор. Инструмент должен звучать как синт (с shape), иначе интуиция не
перенесётся.

**Требуется.**
1. **Ручная маска мод (главное).** Каждый столбик-мода в спектре кликабелен → вкл/выкл. Звук и
   все панели (конвейер, таблица, спектр) пересчитываются по выбранной маске мгновенно.
   - Кандидаты — звучащие моды формы (`modeFreqs` после guard), показывать с потолком ~20.
   - Стартовая маска = текущее поведение (первые `n` включены при spread=0) → старт совпадает с
     синтом.
   - **Нормировка амплитуд (max=1) и shape-проекции — ТОЛЬКО по включённым модам** (паритет с
     тем, как синт нормирует выбранное подмножество).
   - **Низшая ВКЛЮЧЁННАЯ мода = f0** (как `sel[0]` в ядре): выключение самой низшей делает f0
     следующей включённой, спектр перемасштабируется. Привязка к ноте держится на любой маске.
   - `spread` оставить как генератор стартовой маски (двинул → галочки расставились децимацией,
     дальше правишь руками) ЛИБО спрятать в этом режиме — на твоё усмотрение; главный механизм =
     ручные галочки.
2. **Слайдер `shape ∈ [0,1]`** (паритет со звуком синта). Как в `casynth_core.map_laplacian`:
   `amps = (1-shape)*rolloff + shape*proj`, `proj_k = |⟨diag(L), φ_k⟩|` по включённым модам;
   нормировка max=1 после blend. Собств. векторы уже есть (`jacobiEigen` → `res.vectors`). Дефолт 0.
3. **Убрать интерактивный блок «Матрицы формы»** — canvas `#mat` + радио A/D/L + hover-логика и
   связанный JS (`drawMatrix`, mousemove/mouseleave-листенеры, `currentMatType`, `matCells`,
   `DEFAULT_MATHINT`, вызов в `redrawAll`). **Статичную инфографику Части 1 (шаг 3, blinker
   L=D−A) НЕ трогать** — это обучение, а не контрол.

**Критерии приёмки.**
- Клик по столбику-моде вкл/выкл; звук/спектр/таблица/конвейер пересчитываются мгновенно.
- `shape=0` → амплитуды бит-в-бит как сейчас (rolloff `1/i^α`). `shape>0` двигает амплитуды;
  вектор берётся для ТОЙ ЖЕ моды, что дала частоту (выравнивание индексов как в ядре).
- Нормировка и проекции считаются по ВКЛЮЧЁННЫМ модам; низшая включённая нормируется к f0.
- Алгоритм частот/масштаба (`computeLaplacian` pipeline) не тронут — `shape`/маска влияют только
  на выбор мод и амплитуды, не на √λ→Гц.
- Интерактивный matrix-блок удалён; консоль без ошибок; остальные панели работают.

**Default choice (если правки не будет):** инструмента нет — алгоритм автоотбора выбираем
вслепую, без слуховых данных.

## [OPEN] 2026-06-21 — REQ explainer: секция «Краевое возбуждение → формозависимые амплитуды» (4 панели)
_Реквест Researcher → Developer. Дизайн — `decisions.md` 2026-06-18 (краевое возбуждение `e=deg`) +
эта сессия. Файл: `laplacian_explainer.html` (обучалка; JS дублирует map_laplacian — известный
trade-off). Новая обучающая СЕКЦИЯ; переиспользует уже считающиеся `jacobiEigen`→`res.vectors` и
гистограмму мод. НЕ конфликтует с REQ mode-toggle (тот про ВЫБОР мод, этот про ОБЪЯСНЕНИЕ громкостей)
— встают в один файл, можно делать пакетом._

**Цель.** Объяснить на глаз+слух, что такое `e` (краевое возбуждение, `e_i = deg_i` = число живых
8-соседей = диагональ `L`) и как из него рождаются формозависимые амплитуды (`shape`). Стержень:
форма «звучит своей границей» (однородная внутренность `deg=8` нема — ⟂ звучащим модам; звучит
разница `deg` на кромке).

**4 панели (сквозной герой — вытянутая «струна» 2×6; + контрастные пресеты).**
1. **A — «Форма звучит границей».** Клетки раскрашены по `e=deg` (тепло). Переключатель
   `deg ↔ дефицит (8−deg)`: в «дефиците» внутренность гаснет в 0, светится только кромка.
2. **B — «У моды есть ФОРМА».** Клик по столбику-моде → форма раскрашена по `φ_k`
   (red/blue/white = +/−/узел = фигура Хладни); мода играет solo. На «струне» моды = гармоники
   (ниже = плавный градиент, выше = частые чередования).
3. **C — «Громкость = перекрытие удара с модой».** Рядом `e` и `φ_k`; ПОКЛЕТОЧНОЕ произведение
   `e_i·φ_k(i)` раскраской (совпали по знаку = вклад, противоположны = гасят); сумма = `⟨e,φ_k⟩`
   → громкость (число). Слайдер `shape 0→1`: столбик моды переезжает с `1/i^α` на проекцию.
4. **D — «Почему симметрия глушит моду».** Пресет blinker: `e=[1,2,1]` симметричен,
   антисимметричная `φ` (λ=1) → произведение `+0−` суммируется в 0 → столбик = 0 (показать и
   ненормированную сумму=0, и нормированный 0). Пресет «blinker+1» (сломана симметрия) → мода
   оживает. «Симметричная пластина не звенит антисимметричной модой».

**Наглядные/«наслушные» пресеты (кнопки «загрузить форму»; все панели пересчитываются).**
- **«Компактный блок» (3×4)** и **«Протяжённая змейка» РАВНОЙ массы (12 клеток)** — пара для A:
  одинаковая площадь, разная граница → разный паттерн возбуждения. (Как звучит — Пользователь
  оценивает на слух; перцептивное предсказание в критерий НЕ зашивать.)
- **«Струна» (2×6 / линия)** — сквозной герой B+C: моды-гармоники с явными узлами; краевой удар на
  концах → пучности низшей моды (громко) vs узлы иных (тихо).
- **«Blinker»** и **«Blinker+1»** — пара для D.

**Критерии приёмки.**
- Панели пересчитываются по выбранному пресету и выбранной моде; раскраски корректны: `deg`/«дефицит»,
  знак `φ_k` (узлы≈0), знак поклеточного произведения.
- `shape`-проекция в C — БИТ-в-бит как в ядре/mode-toggle (`e=diag(L)`, выравнивание векторов по той
  же моде, что дала частоту, нормировка max=1 по включённым). `shape=0` → столбики = `1/i^α`.
- **Математически проверяемо:** blinker глушит антисимметричную моду (ненорм. сумма `⟨e,φ⟩=0`,
  нормированный столбик = 0); «blinker+1» — та мода ненулевая. (Это и есть тест выравнивания.)
- `e` у «компактного» и «протяжённого» пресетов ВИДИМО различны на форме (паттерн разный);
  слуховой контраст — гипотеза для Пользователя, НЕ критерий кода.
- Solo-проигрыш моды на её частоте; статичную инфографику Части 1 (blinker `L=D−A`) НЕ трогать;
  консоль без ошибок; mode-toggle и прочие панели работают.

**Не делать (out of scope).**
- Не привязывать к ЖИВОМУ полю/динамике (`e(t)`, переливы во времени — отдельная развилка
  «возбуждение как модуляция», в работу не взята).
- Не вводить альтернативные возбуждения (центр.гаусс/радиальное) — секция объясняет ТЕКУЩЕЕ `e=deg`.
- Не менять алгоритм частот/масштаба (`computeLaplacian`) — панели влияют только на показ.

**Default choice (если правки не будет):** `e` остаётся «таинственным» — механизм `shape` непонятен,
дизайн возбуждений ведём вслепую.

## [ANSWERED] 2026-06-18 — REQ Шаг B: формозависимые амплитуды Лапласиана (ручка `shape`)
_Реквест Researcher → Developer. Дизайн-обоснование — `decisions.md` 2026-06-18.
Только `casynth_core.py` (UI-слайдер деривируется из реестра автоматически)._

**Проблема.** Амплитуды `map_laplacian` (`1/i^α`) зависят ТОЛЬКО от номера моды → формы
с близкими низшими собственными значениями звучат одинаково (диагноз 2026-06-16: крупные
формы неразличимы). Нужно сделать громкости мод зависящими от ФОРМЫ, не трогая частоты.

**Требуется (P0 — ядро `map_laplacian`).**
1. **Новый параметр** `shape: float = 0.0` в сигнатуре:
   `map_laplacian(patch, f0, n=N_PARTIALS_DEFAULT, spread=0.0, alpha=1.0, shape=0.0)`.
2. **Собственные ВЕКТОРы.** Заменить `eigvalsh(L)` на `eigh(L)` — нужны и значения, и
   векторы. (Перф: ветвить — при `shape == 0.0` оставить текущий путь `eigvalsh` ровно
   как есть, чтобы гарантировать бит-в-бит и не регрессировать скорость; `eigh` считать
   только при `shape > 0.0`.)
3. **Возбуждение `e` = краевое (graph-degree):** `e_i = deg_i` = число живых 8-соседей
   узла i = диагональ L (`A.sum(axis=1)` по живым узлам, в ТОМ ЖЕ порядке `live`, что и
   строки/столбцы L и собств. векторы). Без внешней нормировки (амплитуды нормируются ниже).
4. **Формозависимые амплитуды:** для каждой звучащей моды k — `proj_k = |⟨e, φ_k⟩|`, где
   φ_k — собственный вектор той же моды, что дала частоту `mode_freqs[k]`. Нормировка
   `proj /= proj.max()` (как существующий `_norm`).
5. **Blend (линейный кросс-фейд):** итоговые амплитуды
   `amps = (1 - shape) * rolloff + shape * proj`, где `rolloff` = текущий `1/i^α`-вектор.
   После blend — нормировать max=1 (сохранить существующую конвенцию и headroom-поведение).
6. **Реестр.** Добавить спек параметра в Laplacian-движок `ENGINES`:
   `('shape', 'shape', 0.0, 1.0, False, 0.0)`. (UI-крутилка появится сама из динамического
   механизма `gol_synth.py` — отдельной правки прототипа не требуется.)

**Критерии приёмки (P0).**
- **A. shape=0 → бит-в-бит.** Существующий `test_laplacian_knob_defaults_backward_compatible`
  и `test_laplacian_amps_nonincreasing_rolloff` проходят БЕЗ изменений (дефолт shape=0 =
  историческое поведение). Не править эти тесты.
- **B. Частоты неприкосновенны.** `shape` НЕ влияет на `freqs` ни на одном значении: при
  любом shape `freqs` идентичны вызову с тем же spread и shape=0. Низшая выбранная мода
  остаётся = f0 (`test_laplacian_lowest_mode_is_f0` проходит при любом shape).
- **C. Выравнивание индексов — критично.** Собственные векторы должны пройти ТЕ ЖЕ маски и
  тот же выбор, что и частоты, и в том же порядке: (1) отбрасывание нулевой моды
  (`eigs > 1e-6`), (2) анти-алиас-гвард (`mode_freqs < _GUARD`), (3) `_select_modes(sel)`.
  После всех трёх столбец j матрицы векторов обязан соответствовать `mode_freqs[j]`.
  Рекомендация: нести индексный массив переживших мод сквозь все маски/выбор и им
  индексировать столбцы `eigh`-матрицы — НЕ пересортировывать векторы отдельно.
  Acceptance-тест: на blinker (P3, моды {0,1,3}) при shape=1 две амплитуды соответствуют
  проекциям на векторы мод λ=1 и λ=3 — амплитуды конечны, не NaN, max=1.
  **Researcher (2026-06-18): нули допустимы; тест C сверяй ТОЧНЫЕ значения.** На P3 проекция
  на λ=1 строго = 0 (симметричное `e=[1,2,1]` ⊥ антисимметричному `[1,0,−1]`), на λ=3 =
  2/√6 → после нормировки на max ожидаемо `amps[:2] == [0.0, 1.0]`. Ассерть именно это
  (atol 1e-6), НЕ только «нет NaN + max=1»: точные значения ловят перестановку столбцов
  собств. векторов (главный риск выравнивания) — при перепутанном порядке выйдет `[1,0]`,
  что слабая проверка не отличит. Нуль на λ=1 — ожидаемое физически корректное поведение
  симметричной формы, `e=deg` НЕ менять. Обоснование — decisions.md 2026-06-18
  «Нулевые проекции на симметричных формах допустимы».
- **D. Инвариантность амплитуд (асимметричная форма).** Новый тест: на L-форме
  (`_ell_patch`, невырожденный спектр) при shape=1 отсортированный вектор амплитуд
  совпадает (atol 1e-6) для формы и её rot90/rot180/fliplr/flipud/roll. (Краевое
  возбуждение graph-интринсик → должно держаться. На симметричных формах с вырожденными
  λ инвариантность per-mode может слегка нарушаться — это известно и допустимо, см.
  decisions.md; тест ставить на асимметричной форме.)
- **E. Контракт.** `test_contract_shapes_and_ranges`, `test_empty_patch_is_silent`,
  `test_engine_registry_integrity`, `test_engine_registry_call_with_defaults` — зелёные
  (amps∈[0,1], длина n, пустой patch молчит, реестр вызывается с новым дефолтом).

**Не делать (out of scope — отдельные развилки, решаем на слух ПОСЛЕ прослушивания B).**
- Не добавлять альтернативные возбуждения (центральный гауссов / радиальное) — только если
  краевое под прослушиванием не разделит формы.
- Не трогать `extract`/окно 8×8 (развилка флудфилл-графа — отдельный OPEN-тред).
- Не пулить энергию вырожденных мод (фикс «мерцания» на симметричных формах — отложен).
- `shape` применяется ТОЛЬКО к Лапласиану; гармонические движки (FFT/Walsh/Random/Granulo)
  не затрагиваются.

**Default choice (если правки не будет):** фичи нет, шаг B не прослушать.
