# 2026-06-21 — ADSR-огибающая (synth-wide) в `gol_synth.py`

## Запрос (Пользователь)
Был только `release`. Добавить `attack` → довести до классического **ADSR** + sustain-level,
**отдельным UI-блоком** (отдельно от ручек движка озвучки). Модель Пользователя: sustain
длится всё время, пока поколение автомата живо; «decay→релиз сразу» = без сустейна (щипок),
«держать sustain всю жизнь» = полный пэд.

## Решение (как реализовано)
Движок не трекает идентичность объектов → «нота on/off» отсутствует. «Событие ноты» =
**топологическое событие моды**: рождение/смена частоты (on) и смерть/смена (off, уже была
release). ADSR привязан к жизни моды; огибающая = нормированный 0..1 множитель на амплитуду
моды из формы (`amp_new`).

- **A** (ms): level 0→1 (пик) на рождении; **D** (ms): 1→sustain; **S** (0..1): держится всю
  жизнь моды; **R** (ms): прежний tail-пул на смерти/смене (без изменений).
- Реализация в `SlotPool`: `_env_phase` (0 idle/1 attack/2 decay/3 sustain), `_env_level`,
  метод `_advance_env(a, attack_chunks, decay_chunks, sustain)` — один шаг на аудио-чанк
  (каденс `update()` = раз/чанк, как у release; гранулярность ~90ms = CHUNK_S). В Phase 1
  обе ветки (stable / freq-changed) считают `amp_tgt = _env_level·amp_new`; смена частоты
  ретриггерит фазу=attack, level=0. `render_chunk` НЕ тронут (он и так сглаживает cur→tgt).
- `feed_audio`: `attack_chunks/decay_chunks = max(1, round(ms/1000/CHUNK_S))`, `sustain` —
  level. Проброшены в `pool.update(...)` (сигнатура расширена на 3 арг).
- `state`: +`attack_ms/decay_ms/sustain`. Константы `ATTACK_MS_*`/`DECAY_MS_*`/`SUSTAIN_*`
  рядом с `RELEASE_MS_*`.
- UI: `rebuild_ctrls` строит движковые ручки (верх) + **фиксированный блок ENV** A/D/S/R
  (`block='env'`, scope='synth') ниже — позиция фиксирована (не зависит от числа ручек
  движка, макс 4 у Laplace), с заголовком «ENV» + сепаратором. `TOOLBAR_H` 148→236 (окно
  выше ~на 88px), чтобы блок не налезал на полосу тэгов движков. `_fmt_for(integer, is_ms)`
  (переименован is_release→is_ms; sustain форматируется `.2f`).

## Бэк-совместимость
Дефолты A→1 чанк, D=0, **S=1** → `level≡1` всю жизнь моды → `amp_tgt=amp_new` бит-в-бит как
до изменения (decay без эффекта при пике==sustain). Прослушивание `shape` не затронуто.

## Проверка
`artifacts/_adsr_check.py` (gitignored): дефолт=amp_new и держит sustain; A=4/D=2/S=0.5 →
`[0.25,0.5,0.75,1.0,0.75,0.5,0.5…]`; щипок S=0 → гаснет в 0; смена частоты ретриггерит attack
(level→0.25, phase=1). `py_compile` OK. Headless-смоук `main()` 5с (SDL dummy) — exit 124 без
трейсбэка (UI-блок ENV рендерится).

## Логирование контролов в лог сессии (по запросу Пользователя)
**Правило (durable):** ЛЮБОЙ контрол синта пишется в дебаг-лог сессии, чтобы любой баг был
воспроизводим по записи и фикс провалидирован. Старая float-матрица `rec['replay']`
(фикс-ширина, контракт laplacian-пробы) молча теряла `shape` и потеряла бы ADSR.
**Сделано:** добавлен **авторитетный пер-кадровый словарь** `rec['replay_controls']` —
копирует ВЕСЬ control-state (`dict(_ep)` всех движковых ручек + synth-wide note/vol/A/D/S/R +
engine id). Авто-захватывает любую ручку, включая будущие, без правок по одной. Сохраняется
в npz как object-массив (`replay_controls`); читать `np.load(..., allow_pickle=True)`. Легаси
`replay` оставлен нетронутым (laplacian-проба). Проверено: py_compile OK + round-trip npz
(shape/ADSR/все ключи сохраняются и читаются).

## Замечания / known limits
- `_render_probe.py` импортирует `gol_life_synth_laplacian` (предшественник, не тронут) → его
  5-арг `pool.update` валиден; проба НЕ сломана. Она читает легаси `replay` (laplacian); для
  реплея gol_synth-сессий с полным control-state потребитель должен читать `replay_controls`
  (consumer-сторона ещё не написана — есть только запись лога).
- Гранулярность огибающей ~90ms (CHUNK_S) — наследное свойство (как у release).
