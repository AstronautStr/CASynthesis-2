# 2026-06-24 — Спектр-полоска в UI (столбики мод движка)

## Задача
Пользователь: показать в интерфейсе спектр сигнала синта (скетч — пустая зона тулбара
под крутилками движка, справа). Развилка «тип дисплея» вынесена в AskUserQuestion
(анализатор FFT выхода / спектрограмма время×частота / столбики мод движка).
**Выбор Пользователя: столбики мод движка** — точные моды до огибающей.

## Что сделано
Read-only визуализация, без потоков/FFT/захвата аудио — данные `voices` (freqs/amps)
уже есть в draw-потоке (`rt.voices`/`rt.color`).

- `casynth_config.py`: `SPEC_F_MIN=20.0`, `SPEC_F_MAX=20000.0` (+ в `__all__`) — лог-диапазон
  оси X (чистый дисплей, на звук не влияет).
- `gol_synth.py` (`main()`, layout): `_spec_rect = Rect(_ctrl_x, by+140, (W-8)-_ctrl_x,
  (GRID_H*CELL+TOOLBAR_H-4)-(by+140))` ≈ x[666..1032], y[748..822], ~366×74. Размещён
  ниже самой высокой панели крутилок (Laplace, 6 рядов) → не смещается при смене движка.
  Добавлен в `lay` (`spec_rect=`).
- `casynth_ui.py`: новый `_draw_spectrum(screen, small, rect, voices, color, f0_hz)` +
  вызов в `draw_frame` (перед MIDI-баром). Лог-ось X, метки декад 100/1k/10k, тонкий
  маркер несущей f0 (C_EDGE), на каждую моду вертикальная линия высотой `min(amp,1)*h`,
  цвет = `color[voice['label_idx']]` (привязка спектра к объекту поля).

## Проверки
- Аудио golden-master (`artifacts/_golden_master.py`): **PASS** байт-в-байт (звук не тронут).
- Headless dumpframe (`CASYNTH_DUMPFRAME`) + 3-сек live loop: без ошибок.
- `artifacts/_probe_spectrum.py`: реальные голоса из `analyse()` (blinker+8×8 блок+полоса,
  laplacian-дефолты) → 3 голоса×12 мод, бары рисуются корректно (`_probe_spectrum.png`).
