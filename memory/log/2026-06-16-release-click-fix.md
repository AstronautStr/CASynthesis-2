## Диагноз (подтверждён, не угадан)
Снепшот `_session_20260616_130029`. Реплей: клиппинг 0, но граничный |d2| max=4375,
ratio 276× — клики на швах чанков. Контрольный эксперимент (тот же снепшот, форс

## Фикс
`SlotPool`: front/back 2-bank → **active + пул tail-слотов**. Старая мода при смене
частоты бесшовно переносится в tail-слот (копия freq/phase/amp/pan → синус
продолжается), затухает за release_chunks; active рестартует с amp 0. Overlap без
щелчка; хвосты наслаиваются в пэд. `update` теперь берёт phase+pan_cur. Пул исчерпан →
кража самого тихого хвоста (`pool.steals` для диагностики).

Размер пула: замер на снепшоте — 192 не хватает (109 краж громких хвостов 0.61 →
остаточный клик 1065); **N_TAIL=384** → 0 краж, граничный |d2| max 1065→94.5 (пол
короткого релиза), рендер +~5%. >384 без улучшений.

## Итерация 2 — клики при МНОГИХ голосах (снепшот 132446, ~12 голосов)
Тот же баг-класс шире: пул хвостов исчерпывался (2571 кража громких amp~0.57 → клики,
boundary max 3039). Два источника, измерены раздельно на снепшоте:
1. исчерпание пула (огромный пул → 0 краж → max 527);
2. клиппинг (527→53 при низком gain; DEV-1, лечится громкостью/метром).

Фикс: graceful eviction — `_enforce_tail_budget` в начале `update` каждый кадр быстро
(но плавно, FAST_EVICT_CHUNKS) до-гасит самые тихие лишние хвосты, держа TAIL_RESERVE
слотов под бёрст одного шага GOL → громкие кражи не нужны. Параметры (замером):
N_TAIL=768, TAIL_RESERVE=192 (=N_ACTIVE, макс. бёрст), FAST_EVICT_MS=200.

Верификация на снепшоте: steals 2571→0; boundary ratio 21.5→2.25; топ-транзиенты ушли
в интерьер; низкий gain bnd_max=38.5 (пол). Остаток при громкой настройке = клиппинг.
Рендер 3.4× realtime. Регресс: галактика 94.5, penta 1.98, core 14/14, smoke ок.

Под нагрузкой эффективная длина пэда ограничена числом голосов (graceful: старые/тихие
хвосты сбрасываются первыми). Полный 4с-пэд на 12 голосах = ~2300 партиалов = клиппинг
всё равно → это разумный предел. Авто-ограничение release под нагрузку — вопрос дизайна
к Researcher, если понадобится.
