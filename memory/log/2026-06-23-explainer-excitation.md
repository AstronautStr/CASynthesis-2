## Что сделано (REQ-2)
Новый модуль `src/excitation.js` + секция «Часть 1.5» в `index.html` (между теорией и
песочницей) + подключение скрипта (порядок: …→render→**excitation**→audio→app).

**Архитектура (HOW):** секция самодостаточна — собственное состояние (`exForm`/`exSel`/
`exShape`/`exDef`), свой `exBuild()` (граф→L→λ,φ→`e=diag(L)`→звучащие моды), свой solo-audio
(отдельный `AudioContext`), фикс `EX_F0=220`. Переиспользует `jacobiEigen`/`NEIGH`/`GUARD`/`$`.
**НЕ трогает** `computeLaplacian`/`modeMask` песочницы → нулевой риск регресса REQ-1. Все
идентификаторы `ex`-префикс — без коллизий с глобалями песочницы.

**Панели (пресеты: Блок 3×4 / Змейка 12 / Струна 1×8 / Blinker / Blinker+1):**
- **A** — форма в тепле по `e=deg`; тумблер `deg ↔ дефицит (8−deg)` (в дефиците внутренность
  deg=8 → 0, светится кромка).
- **B** — фигура Хладни `φ_k` выбранной моды (red/blue/white = +/−/узел) + кнопка «▶ мода solo»
  (играет моду на её частоте).
- **C** — поклеточно `e·φ_k` (зелёный вклад / малиновый гасит), число Σ⟨e,φ⟩, и столбик моды
  rolloff(`1/iᵅ`) → проекция(`|⟨e,φ⟩|`) по слайдеру `shape`.
- **D** — текст-вывод: симметрия глушит моду (Σ=0 на blinker, оживает на blinker+1).

Выбор моды — кликабельная полоска `#exEig` (общий селектор B/C/D).

**Паритет с ядром:** проекция бит-в-бит формуле `casynth_core.map_laplacian` shape-ветки —
`e=diag(L)`, `proj_k=|⟨e,φ_k⟩|`, выравнивание φ по `nzOffset+rank`, `projN=proj/max(proj)`,
blend `(1-s)*rolloffN+s*projN`, нормировка max=1, α=1, маска=все звучащие. При `shape=0` итог =
`1/iᵅ` (критерий C).
