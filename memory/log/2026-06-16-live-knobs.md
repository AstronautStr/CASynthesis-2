# 2026-06-16 — Живые крутилки тембра Laplace (spread / alpha / release)

Реализация реквеста questions.md «Живые крутилки для итерации тембра Laplace»
(решение decisions.md 2026-06-16: привязка к несущей + рычаги контраста/яркости +
бонус-пэды + мета «конвейер живых крутилок»). Управление — экранные слайдеры мышью
(выбор Пользователя), все три крутилки за один заход.

## Что сделано

**`casynth_core.py`**
- `map_laplacian(patch, f0, n, spread=0.0, alpha=1.0)` — добавлены два параметра,
  обратно-совместимо (дефолты = прежний выход бит-в-бит; стенд зовёт без них).
- Новая чистая функция `_select_modes(n_modes, n, spread)`: spread=0 → n низших мод
  подряд (как было); spread=1 → `linspace(0, n_modes-1, n)` (децимация вверх);
  промежуточные — линейная интерполяция индексов, индекс 0 всегда выбран → низшая
  взятая мода = f0 при любом spread (load-bearing привязка к несущей).
- alpha → спад `1/i^α` (α=1 = старое `1/i`).

**`gol_life_synth_laplacian.py`**
- Константы: `SPREAD_DEFAULT/ALPHA_DEFAULT/ALPHA_MIN/MAX`, `RELEASE_MS_DEFAULT/MIN/MAX`.
  Удалены мёртвые `FADE_MS`/`FADE_CHUNKS`.
- `SlotPool.update(voices, amp_cur, release_chunks)` — длина release теперь live-параметр,
  фиксируется per-slot (`_release_len`) на старте хвоста → поворот крутилки в полёте
  не рескейлит идущий фейд. Атака новой моды осталась 1 чанк (быстрая) → крутится
  только длина хвоста: «удар» ↔ пэд.
- `analyse(grid, f0, spread, alpha)` форвардит крутилки в `map_laplacian`.
- `feed_audio` считает `release_chunks = max(1, round(release_ms/1000/CHUNK_S))`.
- UI: три слайдера (spread/alpha/rel) справа от кнопки паттернов, drag мышью как
  volume, live-readout значений. Хит-зона трека расширена (`inflate(0,14)`).

**`tests/test_casynth_core.py`** +4 теста: backward-compat дефолтов, n=truncation-only
(закрепляет ядро решения 20-vs-8), spread держит f0 и достаёт верх, alpha-яркость
не трогает частоты. 14/14 PASS.
